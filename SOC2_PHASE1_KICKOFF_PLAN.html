### Tuesday–Thursday: Readiness Assessment Execution

**Time: 20 hours total (4–5 hours/day)**  
**Participants:** Compliance Lead, CTO, Ops Lead, Engineering Team (as needed)

**Activities:**

#### Day 1 (Tue): Security Controls Review (CC1–CC7)

- [ ] Meet with CTO (2 hours) — review:
  - Authentication & authorization (JWT, sessions, MFA)
  - Cryptography (AES-256-GCM, TLS, secrets)
  - CSRF protection (middleware, one-time use)
  - Audit logging (immutability, triggers)
  - Incident response procedures (current state)
  - Change management (migrations, code review)

- [ ] Interview engineering team (1 hour) — ask:
  - How do we test access control?
  - How often do we rotate secrets?
  - What happens when someone leaves?
  - How do we handle security incidents?

- [ ] Document findings in `Gap_Analysis.xlsx` — control by control

#### Day 2 (Wed): Availability Controls Review (A1–A2)

- [ ] Meet with Ops Lead (2 hours) — review:
  - Uptime monitoring (current tools, alerting)
  - Backup procedures (frequency, testing, recovery)
  - Disaster recovery plan (RTO/RPO, failover procedures)
  - Database replication (if applicable)
  - On-call rotation

- [ ] Interview finance/procurement (30 min) — ask:
  - What's our current backup vendor + SLA?
  - How often is backup tested?
  - What's acceptable downtime?

- [ ] Review infrastructure (if cloud-managed):
  - Document provider's SLA + compliance responsibilities
  - Identify what we're responsible for vs. provider

- [ ] Document findings in `Gap_Analysis.xlsx`

#### Day 3 (Thu): Confidentiality Controls Review (C1–C2)

- [ ] Meet with CTO + ops (2 hours) — review:
  - Data classification (what's PHI, what's not)
  - Encryption at rest (PII fields, database, backups)
  - Encryption in transit (HTTPS, API calls, email)
  - Data retention & purging (soft-delete, archive)
  - Third-party data sharing (if any)

- [ ] Review patient records (sample):
  - Are sensitive fields actually encrypted?
  - Can we read/decrypt them?
  - What metadata is unencrypted?

- [ ] Document findings in `Gap_Analysis.xlsx`

#### Day 4 (Fri): Gap Scoring + Interviews

- [ ] Score each control gap (impact + effort):
  - **Impact:** High (audit failure) / Medium (finding) / Low (observation)
  - **Effort:** High (4+ weeks) / Medium (2–4 weeks) / Low (<2 weeks)

- [ ] Conduct informal interviews:
  - CEO: Business priorities, budget constraints
  - CTO: Technical feasibility, risk tolerance
  - Finance: Cost sensitivity, timeline pressure
  - Ops: Infrastructure limitations

- [ ] Create preliminary remediation roadmap
  - High-impact, low-effort first (quick wins)
  - Then medium-impact, medium-effort
  - Then stretch goals (nice-to-haves)

---

### Friday: Present Initial Findings

**Time: 1 hour meeting**  
**Participants:** Compliance Lead, CEO, CTO, CFO

**Agenda:**
1. Walk through gap analysis (5 slides)
   - Summary of 24 controls
   - 15 fully implemented ✅
   - 9 with gaps 🟡
   - No critical failures 🔴

2. Highlight quick wins (2–3 gaps that close fast)
   - Example: "Document access control policy" (write once, 20 hours)
   - Example: "Set up uptime monitoring" (tool implementation, 40 hours)

3. Flag risks / blockers
   - Any gaps that require customer coordination?
   - Any infrastructure changes needed?
   - Any timeline pressure?

4. Next week plan: Full readiness assessment report due

**Decision point:** Proceed to Week 2–4 (detailed assessment)?

## Weeks 2–4: Deep Readiness Assessment

### Weekly Activities (4 weeks × 20 hours = 80 hours)

#### Week 2: Detailed Control Testing

For each of the 24 controls, test actual operation:

**Security Controls (CC1–CC7) — 17 controls**
- [ ] Test CSRF token generation, validation, one-time use (run actual curl commands)
- [ ] Test JWT claim validation, expiry, device binding
- [ ] Test role-based access (provider sees only assigned patients, admin sees all, etc.)
- [ ] Test encryption (encrypt/decrypt sample PII fields)
- [ ] Test secrets validation (verify env vars required at startup)
- [ ] Test session revocation (logout → token no longer valid)
- [ ] Test MFA enforcement (account with 2FA required v.s. risk score threshold)
- [ ] Test anomaly detection (run test query that should trigger alert)
- [ ] Test rate limiting (rapid-fire login attempts → blocked)
- [ ] Test audit logging (action → immutable log entry)
- [ ] (+ 7 more controls)

**Availability Controls (A1–A2) — 3 controls**
- [ ] Inspect backup configuration (frequency, destination, encryption)
- [ ] Test backup restoration (actually restore to test DB, measure RTO)
- [ ] Document disaster recovery procedure (if written, is it correct?)

**Confidentiality Controls (C1–C2) — 4 controls**
- [ ] Verify PII fields encrypted in database (check actual data)
- [ ] Verify unencrypted fields acceptable (audit logs, session IDs, etc.)
- [ ] Test TLS (HTTP request → redirects to HTTPS)
- [ ] Review access logs (confirm role-based filtering)

**Document all tests in:** `Control_Test_Results/Week2_Testing_Report.md`

#### Week 3: Policy Gap Analysis

For each of the 8–10 policies needed, assess current state:

| Policy | Current State | Effort | Owner |
|--------|---------------|--------|-------|
| Security Policy | Exists in code comments | 20 hrs | CTO |
| Access Control Policy | RBAC implemented, not written | 15 hrs | Compliance Lead |
| Data Classification | Implicit in encryption | 10 hrs | Compliance Lead |
| Change Management | Git + code review exists | 15 hrs | CTO |
| Risk Management | Informal process | 20 hrs | CTO + Compliance Lead |
| Incident Response | On-call rotation only | 25 hrs | CTO + Ops Lead |
| Backup & Recovery | Procedure exists, not tested | 30 hrs | Ops Lead |
| Key Rotation | Capability built, not scheduled | 20 hrs | CTO + Ops Lead |
| Vendor Management | Informal | 15 hrs | Finance + Compliance Lead |
| **Total** | | **170 hours** | |

**Document all gaps in:** `Readiness_Assessment_Report.md` (Policy section)

#### Week 4: Remediation Roadmap

Create prioritized list of 24 gaps to close:

```
PHASE 2 ROADMAP (Oct–Dec 2026)

QUICK WINS (Weeks 1–4 of Phase 2, Oct 2026)
├─ Write formal security policy (20 hrs, CTO)
├─ Set up uptime monitoring dashboard (15 hrs, Ops)
├─ Create access control policy (15 hrs, Compliance Lead)
└─ Document incident response runbook (25 hrs, CTO + Ops)

CORE WORK (Weeks 5–8 of Phase 2, Nov–Dec 2026)
├─ Document backup & recovery procedures (30 hrs, Ops)
├─ Create disaster recovery runbook (20 hrs, Ops)
├─ Document change management (15 hrs, CTO)
├─ Build risk management procedures (20 hrs, CTO)
└─ Create key rotation schedule + test (20 hrs, CTO)

ONGOING (Jan–Sep 2027)
├─ Monthly evidence collection (10 hrs/month)
├─ Quarterly control testing (20 hrs/quarter)
└─ Monthly policy reviews & updates (5 hrs/month)

TOTAL PHASE 2 EFFORT: 150–190 hours (3–4 weeks full-time)
```

**Document in:** `Remediation_Roadmap.md`

## Weeks 5–8: Auditor Selection + Phase 2 Planning

### Week 5: RFP Process

**Time: 20 hours**

- [ ] Draft RFP document (request for proposal)
- [ ] Identify 3–5 auditor firms (Big Four + mid-size)
- [ ] Send RFP with:
  - Scope definition (3 TSCs)

  - Questions for auditor

### Week 6–7: Auditor Evaluation

**Time: 20 hours**

- [ ] Receive proposals (back from 3–5 auditors)
- [ ] Score on:
  - **Cost** (budget is $15–30K)
  - **Timeline** (can start Oct 1?)
  - **Healthcare experience** (prefer)
  - **References** (call them)
  - **Team experience** (who's leading the audit?)

- [ ] Narrow to top 2 firms
- [ ] Schedule final interviews (CEO, CTO, Compliance Lead)
- [ ] Negotiate SOW (Statement of Work)
  - Scope, timeline, deliverables
  - Fee structure (hourly, fixed, success-based?)
  - Exit clause (what if we discover major issues?)

## Go/No-Go Decision (End of Phase 1, Week 8)

**CEO/CTO/CFO decide: Proceed to Phase 2?**

### Go Criteria ✅
- [ ] Gap count < 10
- [ ] No critical gaps preventing audit
- [ ] Auditor selected + contract signed
- [ ] Phase 2 remediation timeline feasible
- [ ] Budget ($147K–$210K) still approved

### No-Go Criteria ❌
- [ ] Gap count > 15 (too much work)
- [ ] Critical gap impossible to close (e.g., third-party dependency we can't control)
- [ ] Budget cuts require scope reduction
- [ ] Timeline pressure forces delay to 2027

**If Go:** Launch Phase 2 (Oct 1, 2026)  
**If No-Go:** Pause, re-plan, restart in 6 months
